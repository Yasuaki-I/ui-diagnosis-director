## 2. 設計思想（必読・全判断の基準）

### コア価値
**「迷わず・即座に・誰にでも分かる形で出力する。」**

### ターゲットユーザー
Webディレクター 25年経験者の視座を継承：
- **経営層**：結論ファースト・3行で要点・数値の説得力
- **実務層**：明日から実装できる具体性・優先度・工数感
- **外部協力会社**：コピペで使える・編集できる・自社化できる
- **事業会社**：クライアント納品に耐える品質・ブランド一貫性

### 想定利用シーン
エンドユーザーが**社内外の提案・報告に際し、出力スライドを自社PPTXフォーマットにコピペ転記して活用する**。
→ フォントはメイリオ統一（コピペ調和のため）が最重要要件。

## 3. 確定した全方針（Q&A集約）

### 基本仕様
| 項目 | 確定内容 |
|---|---|
| **GPTs商品名** | UI診断ディレクター |
| **フッター著者** | UI/UX診断 by GPTs |
| **フォント** | メイリオ統一（latin/ea/cs すべて Meiryo） |
| **最小フォントサイズ** | 14pt |
| **配色** | NAVY #1C366C / RED #D0021B / ORANGE #F97316 の3色アクセント |
| **背景** | WHITE維持 + 透明度活用で上品さ演出 |
| **キャンバス** | 1280×720 px（16:9） |

### 診断仕様
| 項目 | 確定内容 |
|---|---|
| **10項目評価軸** | 固定（FV/キャッチコピー/CTA設計/信頼性/フォーム/レスポンシブ/読みやすさ/情報設計/ブランド一貫性/表示速度） |
| **スコア配点** | 5段階（5/4/3/2/1点） |
| **総合スコア** | (有効項目合計 ÷ 有効項目数) × 10 = 50点満点 |
| **ランク** | S=45-50 / A=38-44 / B=30-37 / C=20-29 / D=0-19 |
| **改善提案件数** | 5件固定（バナーのみ3件可） |
| **強み・最優先課題** | 各3項目固定 |

### 出力仕様
| 項目 | 確定内容 |
|---|---|
| **標準出力** | 3ファイル（C-1/C-2/C-3） |
| **統合版** | ユーザー明示要求時のみ追加生成 |
| **ファイル名** | `01_UI診断スコアカード_{サービス名}.pptx` / `02_改善提案リスト_{サービス名}.pptx` / `03_ビジュアル診断ボード_{サービス名}.pptx` / `00_UI診断統合レポート_{サービス名}.pptx`（統合版） |
| **入力受付** | URL / スクショ / 両方併用 |
| **対象タイプ判定** | GPTsが自動判定、確認質問なし |
| **N/A判定** | バナー時はフォーム設計/レスポンシブ/読みやすさ/情報設計/表示速度 を `'－'` |

### スライド構成
| ファイル | スライド数 | 内容 |
|---|---:|---|
| C-1 スコアカード | 2枚固定 | サマリ + 詳細スコア表 |
| C-2 改善提案リスト | 1〜2枚自動 | 4件以上=2枚、3件以下=1枚 |
| C-3 ビジュアル診断ボード | 3枚固定 | 構造マップ+総評+行動フロー / スコア★バー視覚化 / Before/After Top3 |
| 統合版 | 6〜7枚 | C-1 + C-2 + C-3 を1ファイルに |

## 7. 絶対遵守の運用ルール

### Instructions J-0「絶対実装ルール」（GPTs側）
- ✅ `03_pptx_builder.py` の関数をそのまま呼ぶ
- ❌ 関数内部の処理を再実装しない
- ❌ 関数呼び出し後にshapeを追加しない
- ❌ スクショ画像を埋め込まない（将来機能）
- ❌ タイトル帯に英語サブタイトルを追加しない
- ❌ 戻り値Presentationに後からスライド/図形を追加しない

### 開発フロー
1. ユーザー：意思決定（複数選択肢から[A]/[B]/[C]選択）
2. 私：実装着手
3. 私：プレビュー報告
4. ユーザー：フィードバック
5. 私：微調整 → ZIP更新
